# 视频高光剪辑 Skill

基于火山引擎 Ark API（多模态理解）和 LAS 算子（智能剪辑），自动识别视频中的高光片段并生成集锦视频。

## 触发条件

当用户消息包含以下关键词时触发：
- 视频高光、高光剪辑、精彩集锦、视频剪辑、剪辑视频
- highlight、highlight reel、video highlight

## 工作流

### Step 0: 输入解析

识别用户提供的视频来源和剪辑需求：

- **视频来源**: 本地文件路径、URL（B站/YouTube）、TOS 路径
- **剪辑需求**: 自然语言描述（如 "剪辑最精彩的 3 分钟"、"只保留进球片段"）
- **输出格式**: 默认 mp4

### Step 1: 视频获取 (`video_fetcher`)

根据来源类型下载/加载视频，进行预处理：

- 格式统一为 mp4（非 mp4 文件通过 FFmpeg 转码）
- 提取音频（16kHz 单声道 wav）
- 关键帧采样（默认每 2 秒一帧）

### Step 2: 高光检测 (`highlight_detector`)

**主路径 — 多模态理解（Ark API）:**
- 将关键帧序列 + ASR 文本发送给豆包多模态模型
- 模型分析画面变化、内容精彩度、节奏感
- 返回结构化 JSON（时间戳 + 标签 + 精彩度评分）

**降级路径 — 规则引擎:**
- Ark API 不可用时自动切换
- 音频分析：音量峰值、语速变化、静音段检测
- 画面分析：场景切换检测、运动强度、亮度变化
- 综合打分 → Top-K 片段输出

### Step 3: 视频剪辑 (`video_editor`)

**主路径 — LAS `las_video_edit` 算子:**
- 组装 task_description（用户需求 + 片段列表 + 转场要求）
- 提交 LAS 任务 → 轮询等待完成 → 获取剪辑结果

**降级路径 — FFmpeg 本地剪辑:**
- LAS 不可用时自动切换
- 逐片段裁剪 → concat 拼接 → 输出集锦视频

### Step 4: 输出

- 集锦视频文件
- 时间戳说明（每个片段的起止时间和精彩度评分）
- 支持 JSON 格式导出

## 环境变量

| 变量 | 说明 | 必需 |
|------|------|------|
| `ARK_API_KEY` | 火山方舟 Ark API Key | 是（多模态路径） |
| `LAS_API_KEY` | LAS 算子 API Key | 是（LAS 剪辑路径） |

## Gotchas

- **长视频处理**: 超过 30 分钟的视频建议先分片处理，避免单次多模态调用上下文溢出
- **B站/YouTube 下载**: 需要安装 yt-dlp，部分平台可能有访问限制
- **LAS 轮询超时**: 默认 600 秒超时，大文件可能需要更长时间
- **FFmpeg 降级**: 降级路径使用本地计算资源，大视频注意磁盘空间
- **关键帧采样间隔**: 默认 2 秒，可通过 `DetectorConfig.frame_interval` 调整
- **API Key 安全**: 不要将 API Key 硬编码在代码中，使用环境变量或 `.env` 文件

## 审查标准

- 高光片段时长合理（建议 2-10 秒/段）
- 集锦总时长不超过原视频的 30%
- 片段之间过渡自然（0.5s 转场）
- 输出视频格式为 mp4（H.264 + AAC）
